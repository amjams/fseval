Resampling can be used to take random samples from the dataset, with- or without replacement. A usecase would be to perform [bootstrapping](https://en.wikipedia.org/wiki/Bootstrapping_(statistics)). This means, that we can run the pipeline multiple times using the same configuration, but with different resamplings of the dataset. In this way, we can measure the [stability](https://www.jmlr.org/papers/volume18/17-514/17-514.pdf) of a feature ranking algorithm.

The resampling config looks as follows:

<CodeBlock className="language-python">{ResampleConfig}</CodeBlock>


## Available resampling methods
Built-in resampling methods are `shuffle` and `bootstrap`.

### Bootstrap
Takes random samples with replacement. By default, uses resamples back to the amount of original dataset samples, using `sample_size=1.00`.

<CodeBlock className="language-yaml" title="conf/resample/bootstrap.yaml">{BootstrapYAML}</CodeBlock>

On the command line, use with `resample=bootstrap`.

### Shuffle
This resampling method shuffles the dataset samples.

<CodeBlock className="language-yaml" title="conf/resample/shuffle.yaml">{ShuffleYAML}</CodeBlock>

On the command line, use with `resample=shuffle`.
